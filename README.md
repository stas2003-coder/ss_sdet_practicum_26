### В проекте были сделаны два тестовых сценария:
  - позитивный сценарий - test_scenario_positive.py
  - негативный сценарий - test_scenario_negative.py


## Заметки и замечания
В негативном сценарии я решил проверить незаполнение обязательного поле ввода 'Name'.
Тоесть по ценарию ожидаемый alert не появляется и система переводит текущий курсор на 
не заполненое пустое поле Name.
Для информации, поведение системы в этом случае не совсем удобно с точки зрения User Experience. 
Поле должно подсвечиваться красным с сообщением типо "Заполните обязательное поле" под строкой ввода. 
А текущее поведение, где курсор переходит на пустое поле может быть не сразу заметен и не понятен соответсвено.
Так же можно было протестировать поле ввода имейла на корректный формат, но в данном случае есть ошибка и
данное поле принимает любой формат, поэтому негативный сценарий тут сделать не получиться.

## Инструкция по установке и запуску проекта

### Клонирование репозитория
```bash
git clone https://github.com/stas2003-coder/ss_sdet_practicum_26.git
cd ss_sdet_practicum_26
```

### Создание виртуального окружения
#### Linux / MacOS
```bash
python3 -m venv venv
source venv/bin/activate
```

#### Windows
```bash
python -m venv venv
venv\Scripts\activate
```


### Установка зависимостей
Установка зависимостей из файла `requirements.txt` находящегося в корне проекта:
```bash
pip install -r requirements.txt
```

### Запуск тестов с генерацией Allure отчета
To run the tests and generate an Allure report, use the following command:
```bash
pytest -m "main" --alluredir=./allure-results
```

### Запуск локального сервера и просмотр Allure отчета
After the tests have been executed, you can generate and view the Allure report with:
```bash
allure serve ./allure-results
```
